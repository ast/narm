#+TITLE: narm — Nina Arvid Radio Manager
#+OPTIONS: toc:2 num:nil

* Overview

=narm= is a CLI that manages channels and setup for several
handheld ham radios from one TOML source of truth. The central
config is "compiled" to each radio's native format, filtering out
unsupported channels and modes per target.

Supported radio targets:

- Wouxun KG-Q336
- Quansheng UV-K5
- TYT MD-380
- AnyTone AT-D878UV
- Yaesu FT-50R

* Install Rust

=narm= requires a recent stable Rust toolchain (edition 2024).
Install via [[https://rustup.rs][rustup]]:

#+begin_src sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
#+end_src

After install, reload your shell or =source "$HOME/.cargo/env"=,
then verify:

#+begin_src sh
rustc --version
cargo --version
#+end_src

To pick up newer toolchain releases later:

#+begin_src sh
rustup update stable
#+end_src

* Build & run

Clone and build a debug binary:

#+begin_src sh
git clone <repo-url> narm
cd narm
cargo build
#+end_src

Run the CLI directly through Cargo:

#+begin_src sh
cargo run -- list-radios
cargo run -- validate samples/sample.toml
cargo run -- compile samples/sample.toml --radio quansheng-uv-k5
cargo run -- completions zsh > _narm
#+end_src

The =<config>= argument may be either a single =.toml= file or
a directory of =*.toml= files. For a directory, every file is
loaded in lexicographic order and the =[[channels]]= arrays are
concatenated — use prefixes like =10-uk.toml=, =20-simplex.toml=
to control channel order. Hidden files (=.foo.toml=) are
skipped; recursion is not done.

Release build, lint, format, and tests:

#+begin_src sh
cargo build --release
cargo fmt
cargo clippy --all-targets -- -D warnings
cargo test
#+end_src

* Subcommands

- =validate <config>= :: Parse and validate a channel config
  (file or directory); reports =OK: N channels= or the first
  error.
- =compile <config> --radio <radio> [--out <file>]= ::
  Compile to a target radio's format. (Per-radio compilers are
  not yet implemented; the surface and validation are in place.)
- =list-radios= :: List supported radios with their ids and
  modes.
- =grid <locator> | <lat> <lng>= :: Convert between Maidenhead
  grid locator and lat/lng. One arg = locator (decode → centre
  of cell, prints =lat lng=); two args = coords (encode → 6-char
  locator). Examples: =narm grid JO67AT=; =narm grid 57.8125
  12.0417=.
- =repeaters import <csv>= :: Import a SSA repeater CSV
  ([[https://www.ssa.se/vushf/repeatrar-fyrar/]]) into the local
  SQLite database (FTS5-indexed on call/city/district/network).
  Re-imports update in place via the stable SSA =id=.
- =repeaters near <locator> | <lat> <lng>= :: List repeaters
  within a radius of a location. Flags: =--radius <km>=
  (default 50), =--band <list>=, =--mode <list>=,
  =--limit <N>= (default: no limit), =--tsv= (pipe-friendly;
  default is an aligned table). =--band= and =--mode= accept
  comma-separated values, repeated flags, or a mix
  (=--band 2,70=, =--band 2 --band 70=, =--band 2,70 --band 23=).
  Examples: =narm repeaters near JO67AT --radius 30 --mode fm=;
  =narm repeaters near 57.7 11.97 --band 2,70 --tsv=.
- =repeaters search <query>= :: Full-text search over =call=,
  =city=, =district=, =network= via FTS5. By default the query
  is treated as free text — whitespace-separated terms ANDed
  together, FTS5 metacharacters (=-=, =:=, =+=, parens) treated
  literally. A trailing =*= on a term still does prefix
  matching (=SK6*= matches =SK6RDG=, =SK6RFQ=, …). So both
  =narm rep s D-Star= and =narm rep s SK6*= just work. Pass
  =--raw= to use FTS5 syntax directly for column filters
  (=call:SA6*=) or boolean (=Göteborg AND call:SK6*=). Same
  =--band=, =--mode=, =--limit=, =--tsv= flags as =near=.
  Examples: =narm rep s Angered=; =narm rep s SK6* --band 70=;
  =narm rep s --raw "call:SK6*"=.
- =completions <shell>= :: Emit shell completions
  (=bash=, =zsh=, =fish=, =elvish=, =powershell=) to stdout.

The repeaters database lives at
=$XDG_DATA_HOME/narm/repeaters.db= by default; override with
=--db <path>= or =NARM_DB= env (loaded via =dotenvy=).

* Crates used

=clap= (derive + env), =clap_complete=, =serde= (derive),
=toml=, =csv=, =rusqlite= (bundled, includes FTS5),
=dirs=, =thiserror=, =anyhow=, =dotenvy=.

* Project layout

- Subcommand-based CLI.
- =commands= module with each subcommand in its own file.
- No =mod.rs=; uses =commands.rs= plus a =commands/= directory.
- =lib.rs= holds everything that is not a command, so
  integration tests can exercise core functionality.

#+begin_example
src/
  lib.rs            core types, validation, errors
  main.rs           dotenvy + Cli::parse + dispatch
  commands.rs       Cli/Command definitions, module declarations
  commands/
    validate.rs
    compile.rs
    list_radios.rs
    completions.rs
samples/
  sample.toml       reference config
tests/
  validate.rs       integration test against the sample
#+end_example

* Channel schema

Channels are TOML tables tagged by =mode=. Mode-specific fields
live on the same table as the common ones
(=#[serde(flatten)]= on the Rust side):

#+begin_src toml
[[channels]]
name = "GB3WE"
rx_hz = 145_725_000
shift_hz = -600_000
power = "low"
scan = true
mode = "fm"
bandwidth = "wide"
tone_tx_hz = 88.5
tone_rx_hz = 88.5

[[channels]]
name = "Simplex 2m"
rx_hz = 145_500_000
mode = "fm"

[[channels]]
name = "DMR-SW"
rx_hz = 439_412_500
shift_hz = -9_400_000
mode = "dmr"
color_code = 1
slot = 2
talkgroup = 23_505
admit = "color_code_free"

[[channels]]
name = "GB7DC"
rx_hz = 439_575_000
shift_hz = -9_000_000
mode = "dstar"
urcall = "CQCQCQ"
rpt1 = "GB7DC  B"
rpt2 = "GB7DC  G"

[[channels]]
name = "Wires-X"
rx_hz = 145_375_000
mode = "c4fm"
dg_id_tx = 0
dg_id_rx = 0
data_rate = "dn"
#+end_src

* Type sketch

#+begin_src rust
#[derive(Deserialize)]
struct Channel {
    name: String,
    rx_hz: u64,
    #[serde(default)] shift_hz: i64,
    #[serde(default)] power: Power,
    #[serde(default)] scan: bool,
    #[serde(flatten)]  mode: Mode,
}

#[derive(Deserialize)]
#[serde(tag = "mode", rename_all = "lowercase")]
enum Mode {
    Fm {
        #[serde(default)] bandwidth: Bandwidth,
        #[serde(default)] tone_tx_hz: Option<f32>,
        #[serde(default)] tone_rx_hz: Option<f32>,
        #[serde(default)] dcs_code:   Option<u16>,
    },
    Dmr   { color_code: u8, slot: u8, talkgroup: u32,
            #[serde(default)] admit: Admit },
    Dstar { urcall: String, rpt1: String, rpt2: String },
    C4fm  { dg_id_tx: u8, dg_id_rx: u8, data_rate: C4fmRate },
    P25   { nac: u16, talkgroup: u32 },
    M17   { destination: String, can: u8 },
}
#+end_src
