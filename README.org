#+TITLE: narm — Nina Arvid Radio Manager
#+OPTIONS: toc:2 num:nil

* Overview

=narm= is a CLI that manages channels and setup for several
handheld ham radios from one TOML source of truth. The central
config is "compiled" to each radio's native format, filtering out
unsupported channels and modes per target.

Supported radio targets:

- Wouxun KG-Q336 — see
  [[file:software/README-wouxun-kg-q336.org][software/README-wouxun-kg-q336.org]]
  for CPS install notes (Wine).
- Quansheng UV-K5 — see
  [[file:software/README-quansheng-uv-k5.org][software/README-quansheng-uv-k5.org]]
  for CPS install notes (Wine).
- TYT MD-380
- AnyTone AT-D878UV
- Yaesu FT-50R

* Install Rust

=narm= requires a recent stable Rust toolchain (edition 2024).
Install via [[https://rustup.rs][rustup]]:

#+begin_src sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
#+end_src

After install, reload your shell or =source "$HOME/.cargo/env"=,
then verify:

#+begin_src sh
rustc --version
cargo --version
#+end_src

To pick up newer toolchain releases later:

#+begin_src sh
rustup update stable
#+end_src

* Build & run

Clone and build a debug binary:

#+begin_src sh
git clone <repo-url> narm
cd narm
cargo build
#+end_src

Run the CLI directly through Cargo:

#+begin_src sh
cargo run -- --list-radios
cargo run -- info samples/sample.toml
cargo run -- encode -R quansheng-uv-k5 -c samples/sample.toml -o uvk5.csv
cargo run -- completions zsh > _narm
#+end_src

The =<config>= argument may be either a single =.toml= file or
a directory of =*.toml= files. For a directory, every file is
loaded in lexicographic order and the =[[channels]]= arrays are
concatenated — use prefixes like =10-uk.toml=, =20-simplex.toml=
to control channel order. Hidden files (=.foo.toml=) are
skipped; recursion is not done.

Release build, lint, format, and tests:

#+begin_src sh
cargo build --release
cargo fmt
cargo clippy --all-targets -- -D warnings
cargo test
#+end_src

* CLI shape

The CLI mirrors [[https://www.mankier.com/1/dmrconf][=dmrconf=]]:
verb-based, with global =[Options]= preceding the verb and an
optional file argument:

#+begin_example
narm [OPTIONS] [COMMAND] [FILE]
#+end_example

** Global options

- =-R, --radio <NAME>= :: Target radio (=wouxun-kg-q336=,
  =quansheng-uv-k5=, =tyt-md-380=, =anytone-at-d878uv=,
  =yaesu-ft-50r=). Required for offline encode / decode /
  verify; optional once =detect= can identify a connected
  radio.
- =-D, --device <DEVICE>= :: Serial port the radio is on (e.g.
  =/dev/ttyUSB0=, =COM3=).
- =-t, --toml=, =-y, --yaml=, =-c, --csv=, =-b, --bin=, =-m,
  --manufacturer= :: File-format flag (mutually exclusive). The
  flag describes the *input* file for read/decode/write/verify
  with a file argument, and the *output* file for live read /
  encode. When omitted, the format is inferred from the file
  extension (=.toml=, =.yaml=/=.yml=, =.csv=, =.bin=, =.kg=).
  =-m= / =.kg= is the vendor-native binary (e.g. Wouxun CPS
  =.kg= text wrapper); =-b= / =.bin= is the raw EEPROM image.
- =-o, --out <FILE>= :: Output path; =-= or omitted = stdout.
- =--list-radios= :: Print supported radios and exit.
- =--verbose= :: Verbose output.

** Verbs

- =read [-D <port>] [--from-file <path>] [-t|-y|-c|-b|-m]
  [-o <file>]= :: Read a codeplug. With =-D=, talks to a live
  radio over USB-serial; the format flag selects the *output*
  format (default TOML). With =--from-file=, decodes a saved
  codeplug; the format flag selects the *input* format (default
  inferred from extension), and =-o='s extension picks the
  output (default TOML). Implemented today: live UV-K5 (TOML or
  raw =-b=), live Wouxun KG-Q336 (TOML, raw =-b=, or CPS =-m=
  via internal unscramble + reshape), and offline =--from-file=
  for both radios in any combination of input and output formats
  except KG-Q336 =-m → -b= (no kg-shape → physical inverse yet).
  Examples:
  =narm read -R quansheng-uv-k5 -D /dev/ttyUSB0 -o my-uvk5.toml=;
  =narm read -R quansheng-uv-k5 -D /dev/ttyUSB0 -b -o backup.bin=;
  =narm read -R wouxun-kg-q336 --from-file ~/codeplug.kg -o
  my-q336.toml=;
  =narm read -R wouxun-kg-q336 --from-file backup.bin -o
  out.kg=.
- =write -D <port> -b <file>= :: Upload a raw EEPROM image to
  the radio. Currently only =-R quansheng-uv-k5=; only the
  first 0x1d00 (7424) bytes are ever written, the factory
  calibration block at 0x1d00..0x2000 is preserved
  unconditionally. Input must be either 0x1d00 or 0x2000 bytes
  (calibration tail silently dropped on the latter). The radio
  is reset after write. Example: =narm write -R
  quansheng-uv-k5 -D /dev/ttyUSB0 -b backup.bin=.
- =encode -R <radio> [-y|-c|-b|-m] [-o <file>] <config>= ::
  Encode a TOML config to a programming format. Today only
  =-c= (CHIRP-generic CSV) is implemented — that's the file
  any CHIRP-supported radio can import via *File → Import*.
  =-y= (dmrconf-compatible YAML) and =-b= / =-m= (per-radio
  binary) return "not implemented" with a clear message.
  Channels using modes the target doesn't support (e.g. D-STAR
  on FM-only Yaesu) are skipped with a warning on stderr; DMR /
  C4FM / P25 / M17 are always skipped for CSV because the CHIRP
  CSV format can't represent them. Examples:
  =narm encode -R yaesu-ft-50r -c samples/sample.toml=;
  =narm encode -R anytone-at-d878uv -c -o out.csv
  samples/sample.toml=.
- =decode -R <radio> [-b|-m] [-o <file>] <file>= :: Decode a
  binary codeplug to TOML. Format-flag-driven (or extension
  fallback) — no content sniffing. =-b= = raw EEPROM dump;
  =-m= = vendor file (e.g. Wouxun =.kg=). Implemented for
  KG-Q336 and UV-K5. Examples: =narm decode -R wouxun-kg-q336
  codeplug.kg=; =narm decode -R quansheng-uv-k5 -b backup.bin
  -o uvk5.toml=.
- =info <file>= :: Print a human-readable summary of any file.
  TOML configs report channel count and validation status; =.kg=
  / 32 KiB Q336 raw / 50 KiB =.kg=-shape images print settings,
  VFO state, FM-broadcast presets, scan groups, and channel
  list. Examples: =narm info samples/sample.toml=; =narm info
  ~/codeplug.kg=.
- =detect=, =verify=, =write-db=, =encode-db= :: Stubs that
  match the dmrconf verb surface; return "not implemented yet"
  for now.
- =grid <locator> | <lat> <lng>= :: Convert between Maidenhead
  grid locator and lat/lng. One arg = locator (decode → centre
  of cell, prints =lat lng=); two args = coords (encode → 6-char
  locator). Examples: =narm grid JO67AT=; =narm grid 57.8125
  12.0417=.
- =repeaters import <csv>= :: Import a SSA repeater CSV
  ([[https://www.ssa.se/vushf/repeatrar-fyrar/]]) into the local
  SQLite database (FTS5-indexed on call/city/district/network).
  Re-imports update in place via the stable SSA =id=.
- =repeaters near <locator> | <lat> <lng>= :: List repeaters
  within a radius of a location. Flags: =--radius <km>=
  (default 50), =--band <list>=, =--mode <list>=,
  =--limit <N>= (default: no limit), =--tsv= (pipe-friendly;
  default is an aligned table). =--band= and =--mode= accept
  comma-separated values, repeated flags, or a mix
  (=--band 2,70=, =--band 2 --band 70=, =--band 2,70 --band 23=).
  Examples: =narm repeaters near JO67AT --radius 30 --mode fm=;
  =narm repeaters near 57.7 11.97 --band 2,70 --tsv=.
- =repeaters search <query>= :: Full-text search over =call=,
  =city=, =district=, =network= via FTS5. By default the query
  is treated as free text — whitespace-separated terms ANDed
  together, FTS5 metacharacters (=-=, =:=, =+=, parens) treated
  literally. A trailing =*= on a term still does prefix
  matching (=SK6*= matches =SK6RDG=, =SK6RFQ=, …). So both
  =narm rep s D-Star= and =narm rep s SK6*= just work. Pass
  =--raw= to use FTS5 syntax directly for column filters
  (=call:SA6*=) or boolean (=Göteborg AND call:SK6*=). Same
  =--band=, =--mode=, =--limit=, =--tsv= flags as =near=.
  Examples: =narm rep s Angered=; =narm rep s SK6* --band 70=;
  =narm rep s --raw "call:SK6*"=.
- =completions <shell>= :: Emit shell completions
  (=bash=, =zsh=, =fish=, =elvish=, =powershell=) to stdout.

The repeaters database lives at
=$XDG_DATA_HOME/narm/repeaters.db= by default; override with
=--db <path>= or =NARM_DB= env (loaded via =dotenvy=).

* Project layout

- Subcommand-based CLI.
- =commands= module with each subcommand in its own file.
- No =mod.rs=; uses =commands.rs= plus a =commands/= directory.
- =lib.rs= holds everything that is not a command, so
  integration tests can exercise core functionality.

#+begin_example
src/
  lib.rs            core types, validation, errors
  main.rs           dotenvy + Cli::parse + dispatch
  commands.rs       Cli/Command definitions, module declarations
  commands/
    read.rs         live or --from-file codeplug read
    write.rs        upload raw EEPROM image to radio
    encode.rs       TOML → CSV / YAML / binary
    decode.rs       binary → TOML
    info.rs         summarise any TOML / .kg / raw image
    detect.rs       (stub) probe connected radio
    verify.rs       (stub) compare file vs radio
    write_db.rs     (stub) call-sign DB → radio
    encode_db.rs    (stub) call-sign DB → manufacturer binary
    list_radios.rs  --list-radios implementation
    grid.rs
    repeaters.rs
    completions.rs
    format.rs       Format enum + extension resolver
samples/
  sample.toml       reference config
tests/
  validate.rs       integration test against the sample
#+end_example

* Channel schema

Channels are TOML tables tagged by =mode=. Mode-specific fields
live on the same table as the common ones
(=#[serde(flatten)]= on the Rust side):

#+begin_src toml
[[channels]]
name = "GB3WE"
rx_hz = 145_725_000
shift_hz = -600_000
power = "low"
scan = true
mode = "fm"
bandwidth = "wide"
tone_tx_hz = 88.5
tone_rx_hz = 88.5

[[channels]]
name = "Simplex 2m"
rx_hz = 145_500_000
mode = "fm"

[[channels]]
name = "DMR-SW"
rx_hz = 439_412_500
shift_hz = -9_400_000
mode = "dmr"
color_code = 1
slot = 2
talkgroup = 23_505
admit = "color_code_free"

[[channels]]
name = "GB7DC"
rx_hz = 439_575_000
shift_hz = -9_000_000
mode = "dstar"
urcall = "CQCQCQ"
rpt1 = "GB7DC  B"
rpt2 = "GB7DC  G"

[[channels]]
name = "Wires-X"
rx_hz = 145_375_000
mode = "c4fm"
dg_id_tx = 0
dg_id_rx = 0
data_rate = "dn"
#+end_src

* Type sketch

#+begin_src rust
#[derive(Deserialize)]
struct Channel {
    name: String,
    rx_hz: u64,
    #[serde(default)] shift_hz: i64,
    #[serde(default)] power: Power,
    #[serde(default)] scan: bool,
    #[serde(flatten)]  mode: Mode,
}

#[derive(Deserialize)]
#[serde(tag = "mode", rename_all = "lowercase")]
enum Mode {
    Fm {
        #[serde(default)] bandwidth: Bandwidth,
        #[serde(default)] tone_tx_hz: Option<f32>,
        #[serde(default)] tone_rx_hz: Option<f32>,
        #[serde(default)] dcs_code:   Option<u16>,
    },
    Dmr   { color_code: u8, slot: u8, talkgroup: u32,
            #[serde(default)] admit: Admit },
    Dstar { urcall: String, rpt1: String, rpt2: String },
    C4fm  { dg_id_tx: u8, dg_id_rx: u8, data_rate: C4fmRate },
    P25   { nac: u16, talkgroup: u32 },
    M17   { destination: String, can: u8 },
}
#+end_src
